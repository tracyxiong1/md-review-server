# README Review Plan

This draft explains how contributors should review a Markdown guide before publishing.

## Goals

| Goal            | Why it matters                                                            |
| --------------- | ------------------------------------------------------------------------- |
| Focus comments  | Reviewers should point to exact text, not describe locations from memory. |
| Track revisions | Each round should keep previous comments visible for follow-up.           |
| Compare changes | Readers should see what changed before accepting the new version.         |

## Reviewer flow

1. Open the Markdown file in the local review workbench.
2. Select a sentence or table row that needs clarification.
3. Add a short comment with the requested change.
4. Ask Codex to read open comments and produce the next version.

## Publishing checklist

- Confirm every open comment has a response.
- Compare the current draft against the previous version.
- Keep screenshots focused on the actual review surface.
