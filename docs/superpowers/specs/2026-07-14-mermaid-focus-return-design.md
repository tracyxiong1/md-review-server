# Mermaid 查看器焦点返回设计

## 一、文档范围

本文定义 Mermaid 查看器打开、键盘导航和关闭过程中的焦点管理规则。目标读者为负责 React 组件、样式和测试的研发人员。

当前问题包括三类状态：

1. 鼠标打开查看器后直接按 `Escape`，入口按钮显示蓝色焦点环。
2. 鼠标点击关闭按钮或遮罩后，活动焦点落到 `BODY`。
3. 查看器声明 `aria-modal="true"`，背景页面仍处于可聚焦状态，组件没有焦点循环逻辑。

本次修改不调整查看器尺寸、工具栏布局、缩放、拖动、滚轮和 Mermaid 渲染行为。

## 二、现有实现与根因

`MermaidBlock` 保存查看器的打开方式。`MermaidDiagramViewer` 在挂载后聚焦关闭按钮，并向父组件上报关闭操作的输入方式。

现有关闭规则如下：

- `Escape` 固定上报 `keyboard`。
- 键盘关闭恢复入口焦点。
- 指针关闭不恢复入口焦点。
- 入口的 `:focus-visible` 同时设置背景、文字和 `2px` 蓝色 outline。

该规则把关闭事件来源直接等同于整个查看器会话的交互方式。鼠标打开后直接按 `Escape` 时，会话被归类为键盘路径。指针关闭路径通过跳过焦点恢复规避高亮，关闭按钮卸载后焦点会落到 `BODY`。

## 三、焦点状态模型

查看器维护会话级交互方式：

```ts
type ViewerInputMethod = "keyboard" | "pointer";
```

状态更新规则如下：

1. 打开时使用入口点击事件初始化交互方式。
2. 查看器内发生指针操作时，交互方式更新为 `pointer`。
3. 查看器内发生有效键盘导航或激活操作时，交互方式更新为 `keyboard`。
4. `Escape` 只负责关闭，不单独改变交互方式。
5. 单独按下 `Shift`、`Control`、`Alt` 或 `Meta` 不改变交互方式。

有效键盘交互包括 `Tab`、`Shift+Tab`、`Enter`、空格以及其他会触发查看器控件行为的非修饰键。

关闭回调携带关闭时的会话交互方式。关闭原因可以用于测试和日志表达，但不用于直接决定焦点环：

```ts
interface ViewerCloseEvent {
  reason: "escape" | "close-button" | "backdrop";
  inputMethod: ViewerInputMethod;
}
```

## 四、关闭后的焦点返回

所有关闭路径都把 DOM 焦点恢复到打开当前查看器的放大按钮。恢复调用使用 `focus({ preventScroll: true })`，避免关闭大图时改变文档滚动位置。

可见反馈按关闭时的会话交互方式处理：

| 打开和操作路径             | 关闭方式         | DOM 焦点 | 可见焦点环           |
| -------------------------- | ---------------- | -------- | -------------------- |
| 鼠标打开，未进行键盘导航   | `Escape`         | 返回入口 | 隐藏                 |
| 鼠标打开                   | 鼠标点击关闭按钮 | 返回入口 | 隐藏，保留真实 hover |
| 鼠标打开                   | 鼠标点击遮罩     | 返回入口 | 隐藏                 |
| 键盘打开                   | `Escape`         | 返回入口 | 显示                 |
| 键盘打开                   | 键盘激活关闭按钮 | 返回入口 | 显示                 |
| 鼠标打开，随后使用键盘导航 | `Escape`         | 返回入口 | 显示                 |
| 键盘打开，随后使用鼠标操作 | 任意关闭方式     | 返回入口 | 隐藏                 |

父组件在恢复焦点前提交入口的焦点环抑制状态。焦点恢复在 React 布局阶段执行，避免入口先绘制一次蓝色 outline 再被隐藏。

入口焦点环抑制状态在以下条件下结束：

- 入口失去焦点。
- 用户在入口上开始新的键盘操作。
- 查看器再次以键盘方式打开。

## 五、可见焦点样式

入口按钮的 hover 和键盘 focus 样式拆分处理：

- `:hover` 保留现有背景、文字颜色和透明度变化。
- 未被抑制的 `:focus-visible` 保留现有背景、文字颜色和蓝色 outline。
- 被抑制的返回焦点不应用 focus 专属背景和 outline。
- 鼠标仍停留在入口上时，只显示真实 hover，不显示蓝色 outline。

抑制状态只作用于 Mermaid 放大入口，不修改全局 `:focus-visible` 规则，也不影响查看器工具栏、评论按钮和输入框。

## 六、模态焦点边界

查看器打开后，页面根节点进入 `inert` 状态。查看器通过 portal 保持可交互。关闭或组件卸载时恢复页面根节点原有的 `inert` 状态。

工具栏包含四个可聚焦按钮：缩小、适应窗口、放大和关闭。焦点循环规则如下：

1. 查看器打开后聚焦关闭按钮。
2. 在最后一个可用按钮上按 `Tab`，焦点移动到第一个可用按钮。
3. 在第一个可用按钮上按 `Shift+Tab`，焦点移动到最后一个可用按钮。
4. 中间按钮使用浏览器默认顺序。
5. 禁用按钮不参与首尾焦点计算。

该机制与 `aria-modal="true"` 共同约束模态交互。焦点循环只处理 `Tab` 边界，不覆盖按钮自身的 `Enter`、空格和点击行为。

## 七、组件职责

### 7.1 MermaidBlock

`MermaidBlock` 负责：

- 保存查看器是否打开及初始交互方式。
- 接收关闭事件。
- 在布局阶段恢复入口 DOM 焦点。
- 设置和清理入口焦点环抑制状态。

### 7.2 MermaidDiagramViewer

`MermaidDiagramViewer` 负责：

- 维护查看器会话中的当前交互方式。
- 打开时聚焦关闭按钮。
- 管理页面根节点的 `inert` 状态。
- 约束工具栏焦点循环。
- 上报关闭原因和关闭时的交互方式。

## 八、验证范围

组件测试覆盖以下行为：

1. 指针打开后关闭按钮获得实际焦点，初始焦点环继续被抑制。
2. 指针打开后直接按 `Escape`，入口恢复焦点但带有焦点环抑制标记。
3. 键盘打开后按 `Escape`，入口恢复焦点且不带抑制标记。
4. 指针点击关闭按钮或遮罩后，入口恢复焦点但不显示焦点环。
5. 指针打开后使用 `Tab`，再按 `Escape` 时显示入口焦点环。
6. 键盘打开后发生指针操作，再关闭时隐藏入口焦点环。
7. 入口抑制状态在失焦和新的键盘操作后清除。
8. 查看器挂载时页面根节点进入 `inert`，卸载时恢复原状态。
9. `Tab` 和 `Shift+Tab` 在可用工具栏按钮之间循环。
10. 禁用缩放按钮不参与焦点循环边界计算。

运行时验证覆盖深色和浅色主题，并复查指针打开、键盘打开、三种关闭方式和混合输入方式。项目级验证执行：

- `pnpm test`
- `pnpm lint`
- `pnpm fmt:check`
- `pnpm build`
