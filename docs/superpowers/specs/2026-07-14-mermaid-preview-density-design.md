# Mermaid 正文预览密度设计

## 一、问题范围

当前 Mermaid 正文预览使用两层垂直间距：外层代码块预留 `48px` 顶部操作区和 `16px` 底部 padding，内层图表容器再增加上下各 `16px` padding。

运行时测量结果如下：

- 图表上方总留白约 `64px`。
- 图表下方总留白约 `32px`。
- 外层预览高度比 SVG 与必要边框高约 `96px`。

该间距保证放大按钮不覆盖 SVG，但操作区和内层留白叠加后整体偏高。

## 二、目标布局

采用独立操作区的紧凑方案：

- 外层顶部操作区从 `48px` 调整为 `36px`。
- 外层底部 padding 从 `16px` 调整为 `4px`。
- 内层图表容器上下 padding 从 `16px` 调整为 `8px`。
- 放大按钮顶部定位从 `12px` 调整为 `4px`，保持 `28px` 视觉尺寸。
- 水平 padding 保持 `16px`。

调整后的图表上方留白约 `44px`，下方留白约 `12px`，整体高度减少约 `40px`。按钮与图表之间保留约 `4px` 间隔，图表内容不缩放。

## 三、交互与视觉约束

该改动只影响 Mermaid 正文预览的垂直密度。

以下行为保持不变：

- 放大按钮位于独立操作区，不覆盖 SVG。
- 按钮的默认、悬停、键盘焦点和无障碍名称。
- Mermaid 查看器的打开、缩放、移动和关闭行为。
- 代码块、错误态和非 Mermaid 内容的 padding。
- 浅色与深色主题的颜色、边框与圆角。

## 四、实现边界

`src/styles/markdown.css` 负责全部尺寸调整。组件结构和 Mermaid 渲染逻辑不需要修改。

样式继续使用现有 `--markdown-pre-padding` 覆盖机制，调整范围限定在 `.mermaid-pre`、`.mermaid-container` 和 `.mermaid-expand-button`。

## 五、验证范围

自动化检查覆盖 Mermaid 专用类名和预览结构，确保样式仍限定在 Mermaid 代码块。

运行时验证覆盖：

1. 深色主题下的宽时序图。
2. 浅色主题下的普通流程图。
3. 放大按钮默认、悬停和键盘焦点状态。
4. 图表与按钮无重叠。
5. 非 Mermaid 代码块 padding 不变。

项目级验证包括 `pnpm test`、`pnpm lint`、`pnpm fmt:check` 和 `pnpm build`。
