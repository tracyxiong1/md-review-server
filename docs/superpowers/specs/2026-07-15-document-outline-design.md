# 文档内标题目录设计

## 一、背景与目标

`md-review-server` 当前提供文件树、Markdown 预览和评论导航。长文档缺少结构化导航，用户需要通过滚动查找章节。

本次变更在 Markdown 文档卡片内部增加标题目录。目录展示当前文档的全部标题层级，并提供点击跳转和滚动位置反馈。该能力同时适用于 CLI 单文件模式和目录浏览模式。

## 二、范围

### 2.1 首版能力

- 解析并展示 Markdown 正文中的 `H1` 至 `H6`。
- 保留标题层级关系，通过缩进表达深度。
- 标题目录位于文档卡片内部左侧，与正文共享背景。
- 完整目录使用固定宽度上限；超长标题单行省略，原文通过 `title` 提示提供。
- 文档卡片小于 `520px` 时切换为 `32px` 紧凑标题轨道。每个标题显示一个按 `H1` 至 `H6` 递减的刻度，当前刻度使用蓝色。
- 紧凑轨道在鼠标悬停或键盘聚焦时显示 `Hn + 完整标题` 浮层；浮层限制在可见预览区域内，不进入评论栏。
- 点击目录项后，正文平滑滚动至对应标题。
- 正文滚动时更新当前章节，并让目录中的当前项保持可见。
- 当前项使用蓝色文字和左侧细线表示，不使用整行色块。
- 无标题文档不渲染目录列。
- Diff 模式不渲染目录；切回 Preview 后重新显示。
- 支持深色和浅色主题。

### 2.2 非目标

- 手动收起或展开目录。
- 根据窗口或文档长度自动隐藏目录。
- 调整目录宽度。
- 保存目录展示状态。
- 修改文件树、评论栏或服务端 API。
- 在 Diff 模式中提供标题导航。

首版运行时评审确认三栏同时展开时 `160px` 目录会把正文压缩到不可读宽度，因此采用卡片容器查询进行紧凑降级；手动收起继续不在本次范围内。

## 三、界面结构

`MarkdownPreview` 保留现有应用级布局。Preview 模式下的文档卡片调整为两列：

1. `DocumentOutline`：文档内部的标题导航区。
2. `markdown-document-body`：文档元信息、Markdown 正文、评论标记层和文本选择入口。

目录区和正文区共享 `markdown-content` 的白色或深色面板背景。两者之间使用 `border-secondary` 分隔。目录不增加独立卡片、标题栏、底色或阴影。

目录宽度使用受控固定值，完整模式为 `160px`。目录项保持单行，层级缩进应在 `H6` 情况下仍保留可读文本空间。`.markdown-content` 小于 `520px` 时，通过容器查询把目录列切换为 `32px` 轨道，不依赖浏览器窗口宽度。

`DocumentOutline` 在 `.markdown-reader-scroll` 内使用 sticky 定位。目录高度受当前预览视口约束，内容超出时独立纵向滚动。

## 四、标题模型与标识

标题解析结果使用以下结构：

```ts
interface DocumentHeading {
  id: string;
  text: string;
  level: 1 | 2 | 3 | 4 | 5 | 6;
  line: number;
}
```

标题来源为 `parseMdContent` 返回的正文。解析逻辑应遵循 Markdown AST，避免把代码块中的 `#`、引用内容或普通文本识别为标题。

标题 ID 使用正文行号生成，例如 `markdown-heading-12`。行号在单次渲染中唯一，重复标题文本不会产生 ID 冲突。ReactMarkdown 的标题组件使用同一规则写入 `id` 和现有 `data-line-start` 属性。

目录文本从标题 AST 的可见文本内容提取。链接、强调、行内代码等内联节点保留文本；格式标记不进入目录文案。

## 五、交互流程

### 5.1 点击跳转

目录项使用语义化链接或按钮，并提供可见的键盘焦点状态。触发后执行以下操作：

1. 查找对应标题元素。
2. 在 `.markdown-reader-scroll` 中平滑滚动到标题位置。
3. 将该标题设置为当前章节。

跳转行为不修改 URL，不写入服务端状态，也不影响评论状态。

### 5.2 滚动联动

Preview 模式监听 `.markdown-reader-scroll` 的滚动和尺寸变化。当前章节按标题相对滚动容器顶部的位置计算：

- 已越过顶部判定线的最后一个标题为当前章节。
- 尚未越过第一个标题时，第一个标题为当前章节。
- 文档滚动到底部时，最后一个已越过判定线的标题保持当前状态。

更新当前章节后，目录容器只在当前项超出可视区域时调整自身滚动位置。该过程不应反向移动正文滚动容器。

滚动事件通过 `requestAnimationFrame` 合并计算，组件卸载、文件切换或进入 Diff 模式时清理监听器和待执行帧。

### 5.3 文件与视图切换

文件内容变化后重新解析标题列表，并将当前章节初始化为第一个标题。无标题文档不保留上一个文件的目录或当前项。

进入 Diff 模式时停止目录滚动监听并移除目录结构。切回 Preview 后基于当前正文重新建立标题列表和滚动状态。

### 5.4 紧凑轨道提示

紧凑轨道的整行链接仍是点击和键盘焦点区域，细刻度只负责视觉表达层级；链接的可访问名称同时包含标题文字和 `H1` 至 `H6` 层级。提示浮层使用固定定位并挂载到 `body`，根据触发链接、可见 `.markdown-container`、浏览器视口和评论栏左边界计算位置。正文滚动、目录滚动、窗口尺寸变化和相关容器重排时，通过 `requestAnimationFrame` 合并重算；可用区域不足 `120px × 44px` 时不显示浮层。

浮层允许指针进入和滚动超长标题。指针从刻度移向浮层时使用短延迟保持显示，离开浮层后关闭；键盘用户可按 `Escape` 关闭，并在重新聚焦或悬停时再次打开。

完整目录继续使用原生 `title`；紧凑模式移除原生 `title`，避免与自定义浮层重复。

## 六、代码边界

| 路径                                      | 职责                                                        |
| ----------------------------------------- | ----------------------------------------------------------- |
| `src/lib/extractDocumentHeadings.ts`      | 从 Markdown 正文 AST 中提取标题模型和稳定 ID。              |
| `src/lib/extractDocumentHeadings.test.ts` | 验证标题层级、内联文本、重复标题、代码块和空文档。          |
| `src/components/DocumentOutline.tsx`      | 渲染目录、当前项和点击入口。                                |
| `src/components/DocumentOutline.test.tsx` | 验证层级、截断提示、当前项、点击回调和可访问状态。          |
| `src/components/MarkdownPreview.tsx`      | 组合目录与正文，写入标题 ID，管理正文滚动联动和视图切换。   |
| `src/components/MarkdownPreview.test.tsx` | 验证跳转、滚动联动、无标题、文件切换和 Diff 模式。          |
| `src/styles/markdown.css`                 | 定义卡片内两列布局、sticky 目录、层级缩进、主题和焦点样式。 |

标题解析和目录展示保持独立。`DocumentOutline` 接收标题列表、当前 ID 和跳转回调，不读取 Markdown 原文，也不直接操作正文 DOM。

## 七、兼容性与错误边界

- 标题解析失败时返回空列表，Markdown 正文继续渲染；目录能力不阻断预览。
- 目录项对应的标题元素不存在时，点击操作保持无副作用。
- 评论标记继续锚定 `markdown-document-body` 左边界，避免与目录文本重叠。
- 文本选择仍以 `markdown-document-body` 为容器，目录文本不进入评论选区。
- Mermaid、代码高亮、表格和 Diff 现有渲染逻辑保持不变。
- `prefers-reduced-motion: reduce` 下使用即时滚动，避免强制平滑动画。

## 八、验证计划

### 8.1 自动化测试

- 标题解析：`H1` 至 `H6`、ATX 标题、Setext 标题、重复标题、内联格式、代码块、无标题。
- 目录渲染：层级属性、完整模式 `title`、紧凑模式刻度与自定义浮层、当前项标识、键盘可聚焦入口。
- 响应式边界：`520px` 两侧模式切换、焦点与鼠标交错、滚动和尺寸变化后的浮层重定位、极小空间隐藏。
- 预览联动：点击定位、滚动更新、目录保持当前项可见、文件切换清理。
- 视图边界：无标题不显示目录，Diff 模式隐藏目录，Preview 恢复目录。
- 回归：评论标记和文本选择容器保持在正文区域。

### 8.2 工程检查

- `pnpm test`
- `pnpm build`
- `pnpm lint`
- `pnpm fmt:check`

### 8.3 运行时状态

实现完成后基于 `design-drafts/runtime-baseline/` 验证以下组合：

- 深色与浅色主题。
- 文件树展开与折叠。
- 评论栏展开与折叠。
- 无评论与有评论。
- 无标题、短目录和超过目录视口高度的长目录。
- Preview 与 Diff 切换。

运行时评审记录显示：`665px` 文档卡片使用 `160px` 完整目录；评论展开后的 `427px` 卡片和三栏展开后的 `233px` 卡片使用 `32px` 紧凑轨道。三栏极限状态下正文区域从约 `71px` 提升到约 `199px`（均含正文内边距）。暂不增加手动收起。

## 九、验收标准

1. CLI 单文件模式和目录浏览模式使用同一标题目录实现。
2. 所有 Markdown 标题层级均可见，重复标题可以独立跳转。
3. 点击目录项能够定位对应标题。
4. 正文滚动能够稳定更新当前章节，目录当前项保持可见。
5. 长标题不会扩大目录宽度或换行，完整文案仍可访问。
6. 无标题和 Diff 模式不显示空目录区域。
7. 目录不会进入评论选区，评论标记保持锚定正文边界。
8. 深色和浅色主题保持与现有设计系统一致。
9. 现有测试、构建、Lint 和格式检查通过。
10. 文档卡片小于 `520px` 时自动切换紧凑轨道，提示浮层不越过预览和评论栏边界。
