# Mermaid 查看器焦点反馈设计

## 一、问题范围

当前查看器在打开时主动聚焦关闭按钮，关闭时无条件把焦点恢复到“放大查看”入口。浏览器会把这两次程序化聚焦判定为 `:focus-visible`，鼠标用户因此看到蓝色焦点环。

运行时复现确认了以下问题：

1. 鼠标点击“放大查看”后，关闭按钮出现蓝色焦点环。
2. 鼠标点击关闭按钮后，“放大查看”入口出现蓝色焦点环。
3. 点击遮罩使用相同的关闭链路，会产生相同的入口焦点环。

`Escape` 关闭后的入口焦点环属于键盘反馈，当前行为正确。已处理评论标记在鼠标打开和关闭后不会残留焦点样式，不需要修改。文本输入区域的自动聚焦也属于预期行为。

## 二、交互规则

查看器根据触发事件区分指针与键盘操作。

### 2.1 打开查看器

- 鼠标点击入口：实际焦点仍移入查看器的关闭按钮，但隐藏这一次程序化聚焦产生的焦点环。
- 键盘激活入口：实际焦点移入关闭按钮，并显示现有焦点环。
- 鼠标打开后，一旦用户开始键盘导航，关闭按钮及其他控件恢复正常的 `:focus-visible` 反馈。

### 2.2 关闭查看器

- 鼠标点击关闭按钮：关闭查看器，不主动把焦点恢复到入口。
- 鼠标点击遮罩：关闭查看器，不主动把焦点恢复到入口。
- 按 `Escape`：关闭查看器，把焦点恢复到入口并保留可见焦点环。
- 键盘激活关闭按钮：关闭查看器，把焦点恢复到入口并保留可见焦点环。

该规则让鼠标路径回到普通悬停状态，让键盘路径继续提供明确的当前位置反馈。

## 三、组件边界

`MermaidBlock` 负责记录打开方式，并根据关闭原因决定是否恢复入口焦点。

`MermaidDiagramViewer` 负责：

- 接收初始聚焦是否需要可见反馈的状态。
- 打开时始终聚焦关闭按钮。
- 将关闭原因区分为指针关闭和键盘关闭。
- 在用户开始键盘导航后取消初始焦点环抑制。

关闭回调携带输入方式，避免父组件从关闭后的 DOM 状态反推来源。

## 四、样式约束

现有全局和组件级 `:focus-visible` 规则保持不变。查看器只对“鼠标打开后的第一次程序化聚焦”增加局部抑制状态。

该状态不影响：

- 键盘打开时的关闭按钮焦点环。
- 查看器内通过 `Tab` 移动后的焦点环。
- `Escape` 或键盘关闭后入口的焦点环。
- 其他按钮、评论标记和输入框的焦点样式。

## 五、验证范围

组件测试覆盖以下行为：

1. 鼠标打开后关闭按钮获得实际焦点，但初始焦点反馈被标记为指针来源。
2. 键盘打开后关闭按钮获得实际焦点，并使用正常可见焦点。
3. 鼠标点击关闭按钮不会恢复入口焦点。
4. 点击遮罩不会恢复入口焦点。
5. `Escape` 关闭后恢复入口焦点。
6. 键盘激活关闭按钮后恢复入口焦点。
7. 开始键盘导航后移除鼠标初始焦点的局部抑制状态。

项目级验证包括 `pnpm test`、`pnpm lint`、`pnpm fmt:check` 和 `pnpm build`。运行时复查深色与浅色主题下的鼠标打开、鼠标关闭、遮罩关闭、键盘打开和键盘关闭状态。
