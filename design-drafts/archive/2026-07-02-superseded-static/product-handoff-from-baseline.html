Codex review handoff

The review page collects precise comments on selected Markdown text. Codex reads the open comments, writes the next version, and records the result back into the review sidecar.

Current friction

After commenting in the browser, the user still has to return to the conversation and type a prompt. That prompt is only a trigger signal, so the interface can make the handoff explicit without hiding what will happen.

Proposed behavior

The comments panel owns the handoff state because it already contains the open comment count, file name, and review history. The document remains the primary surface.
