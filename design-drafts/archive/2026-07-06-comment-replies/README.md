# Comment Replies Drafts

These HTML drafts explore the comment reply interaction for mixed review comments.

- [v1-inline-thread.html](./v1-inline-thread.html): replies live inside each comment, with an inline user reply editor.
- [v2-confirmation-section.html](./v2-confirmation-section.html): open comments are grouped so confirmation blockers appear first.
- [v3-latest-carryover-light.html](./v3-latest-carryover-light.html): latest-version carryover in light mode with expanded sidebar.
- [v4-open-order-no-extra-status.html](./v4-open-order-no-extra-status.html): refined direction, no extra confirmation status, comments stay in document order, but changes too much of the baseline panel.
- [v5-baseline-plus-replies.html](./v5-baseline-plus-replies.html): baseline-aligned direction. It starts from runtime HTML and only adds inline reply display plus the active comment reply editor, but the inner reply card feels too nested.
- [v6-flat-message-thread.html](./v6-flat-message-thread.html): compact flat message thread inside each comment item, but the avatar marks make it feel too much like chat UI.
- [v7-author-label-thread.html](./v7-author-label-thread.html): preferred direction. It keeps only `你` and `Codex` author labels, with no avatars and no nested reply cards.
- [v8-current-implementation-preview.html](./v8-current-implementation-preview.html): preview of the current implemented UI details, including border-centered markers, `Codex`/`你` author labels, and relative/absolute reply times.

Preferred direction to evaluate first: `v8-current-implementation-preview.html`.
