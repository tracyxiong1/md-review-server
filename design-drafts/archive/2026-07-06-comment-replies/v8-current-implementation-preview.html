Event Pipeline Observability

1. Goals

  • Detect ingest interruption within two minutes.
  • Redesign event schema.
  • Keep replay context stable.

2. Signal Model

Each stage exports common metric dimensions:

  • tenant_id
  • region
  • event_type
  • pipeline_stage

3. Delivery Health

The dashboard groups received, rejected, processing, and delivered counts by stage. Operators should be able to follow a single event family from gateway validation to consumer publish without switching tools.
