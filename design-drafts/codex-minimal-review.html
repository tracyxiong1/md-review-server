Browser Automation and Agent Mechanisms

本文档说明浏览器自动化能力在 Agent 系统中的定位，并解释工具层、协议层和执行层之间的边界。

Agent 与工具层边界

工具层负责封装浏览器控制协议与执行细节。Agent 负责决策、规划和结果校验，避免把流程逻辑沉到工具层。

CDP、WebDriver 与 WebDriver BiDi 分别覆盖不同的接入边界。选择协议时，应优先考虑稳定性、权限边界和跨浏览器需求。

适用场景

适合端到端验证、受控浏览器操作和需要页面状态回读的任务。对于只需要纯文本处理的工作，浏览器自动化通常不是首选。

在涉及登录态、敏感权限或本地文件访问时，需要把授权边界写清楚。 这类信息会影响用户是否应该选择浏览器自动化方案。

与 Codex Review Loop 的关系
